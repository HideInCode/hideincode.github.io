2026-05-01 算法 algo 0 评论

用于提升解题速度

复杂度要求

数据规模 N 最大可接受时间复杂度 常见算法示例
≤ 10 O(N!) 全排列（回溯）
≤ 20 O(2^N) 子集枚举、状态压缩 DP
≤ 100 O(N^3) Floyd-Warshall、区间 DP、三重循环
≤ 500 O(N^3) 勉强 某些 DP（较小常数）
≤ 2,000 O(N^2) 冒泡、插入、简单 DP（如 LIS 朴素版）
≤ 10^4 O(N√N) 或 O(N log^2 N) 分块、莫队算法
≤ 10^5 O(N log N) 快排、归并、线段树、树状数组、Dijkstra、二分答案
≤ 10^6 O(N log N) 或 O(N) 线性筛、双指针、前缀和、单调栈、BFS/DFS
≤ 10^7 O(N) 或 O(N log log N) 埃氏筛、线性 DP、桶排序
≤ 10^8 严格 O(N)，常数极小 简单遍历（一般较少）
> 10^8 O(1) 或 O(log N) 公式计算、数学推导、二分法

补充说明

  1. 常数因子很重要
    O(N log N) 在 N=10^5 秒过，但 N=10^6 时若操作复杂（如 map/set 大量插入）可能超时。Python 通常会比 C++ 慢 10~50 倍，需保守估计：一般认为 Python 每秒运算约 5×10^7 次，C++ 约 2×10^8 ~ 5×10^8 次。
  2. 内存限制
    通常为 256MB：
    • int[N]（N=10^6）≈ 4MB（C++）/ 28MB（Python 对象开销大）
    • int[N][N]（N=2000）≈ 4MB（C++）但在 Python 中极易爆内存，需用 list 慎重，或改用其他结构。
  3. 特殊场景
    • 如果有多组测试用例，需考虑总复杂度。假设 T=10^5，总复杂度必须 ≤ O(N) 或 O(N log N) 且 N 为所有用例数据总和。
    • 递归深度：Python 默认递归上限 1000，DFS 图/树时需要改写成栈，或 sys.setrecursionlimit(1000000)。

掌握这些对应关系后，你就可以快速判断一道题该用什么算法。

题目特征

1. 双指针 / 滑动窗口

  • 适用特征：数组/字符串，求满足某种条件的子数组/子串，或要求原地修改。
  • 思考路径：
    • 是否需要两端收缩？→ 左右指针
    • 是否需要维护一个窗口满足条件？→ 滑动窗口
    • 是否需要快慢步调？→ 快慢指针（环检测、找中点）

2. 二分查找

  • 适用特征：数据有序（或可以构造出有序性），求目标值、边界，或答案值域连续可判定。
  • 思考路径：
    • 基本二分：在有序数组中找目标。
    • 二分答案：问题可以转化为“对于某个 x，是否能满足条件？”（最小化最大值/最大化最小值/第K小）

3. 动态规划

  • 适用特征：求最优解、总数、路径数，且大问题可分解为重叠子问题。
  • 思考路径：
    • 一维 DP：当前状态只与前一两个状态有关（爬楼梯、打家劫舍）
    • 二维 DP：两个序列/字符串（最长公共子序列、编辑距离）
    • 背包模型：物品 + 容量限制（分割等和子集、零钱兑换）
    • 区间 DP：回文、石子合并
    • 状态机 DP：买卖股票系列

4. DFS / BFS（搜索与回溯）

  • 适用特征：求所有方案、路径、连通性，或图/树的遍历。
  • 思考路径：
    • BFS：最短路径、最短步数、层序遍历 → 用队列
    • DFS：所有排列/组合、连通块、树的后序遍历 → 递归/栈
    • 回溯：路径探索 + 撤销选择（全排列、N皇后）

5. 贪心

  • 适用特征：最优解可通过局部最优累积得到，具有贪心选择性质（区间调度、跳跃游戏）。
  • 思考路径：排序 + 按某种优先级选取。通常需要证明或感觉可行（先尝试贪心，失败再转 DP）。

6. 哈希表

  • 适用特征：快速查找、去重、计数、映射关系。
  • 思考路径：两数之和（找互补）、最长连续序列（set 快速判断邻居）、前缀和 + 哈希（和为K的子数组个数）

7. 栈 / 单调栈

  • 适用特征：括号匹配、表达式求值、找下一个更大/更小元素、区间边界问题。
  • 思考路径：单调递增/递减栈能快速确定以某个元素为最值的区间（柱状图最大矩形、接雨水、每日温度）

8. 数学与位运算

  • 适用特征：数字规律、幂运算、唯一出现、二进制状态压缩。
  • 思考路径：快速幂、异或性质、摩尔投票、素数筛、最大公约数。

常用技巧（优化代码和性能）

1. 降低时间复杂度的编码技巧

  • 用空间换时间：提前存储中间计算结果（记忆化搜索、前缀和、哈希缓存）
  • 二分代替线性查找：在有序或可判定序列中快速定位边界
  • 双指针降维：将 O(n²) 的遍历变为 O(n)（如三数之和固定一端后双指针）
  • 位运算加速：n & (n-1) 去掉最后一位 1，x & -x 取 lowbit，快速判断奇偶 n & 1

2. 边界处理技巧

  • 虚拟头节点：链表操作避免判空
  • 数组长度加 1：前缀和方便计算区间和
  • 哨兵/填充：二维网格四周填充特殊值减少越界判断

3. 代码简化技巧

  • 使用 Python 特性：collections.Counter 计数，defaultdict 省去初始化判断，enumerate 同时取索引和值
  • 列表推导式快速生成数组
  • @lru_cache 自动记忆化 DFS
  • heapq 快速实现优先队列

4. 调试与验证

  • 手写小样例推演：遇到卡壳时用笔在纸上画出 dp 表或树图
  • 打印中间状态：关键位置 print 检查变量是否符合预期
  • 对拍：写个暴力解和优化解对比随机小数据

三、如何提高做题速度

1. 建立题型反射弧（模式识别）

  • 按之前整理的知识分类刷题，每种类型刷 10~20 道，直到看到题目描述就能条件反射出解法模型。
  • 关键词映射表（部分）：
    • “最长公共子序列” → 二维 DP
    • “子数组和为 K” → 前缀和 + 哈希
    • “下一个更大元素” → 单调栈
    • “所有可能的组合” → 回溯
    • “最短路径” → BFS
    • “最大值最小/最小值最大” → 二分答案

2. 精通常用模板（达到闭眼能写）

  • 至少熟练以下模板，无需重新思考边界：
    • 二分查找（左闭右闭/左闭右开）
    • 回溯框架（路径、选择列表、撤销）
    • 01背包/完全背包（一维优化正逆序）
    • BFS 层序遍历（队列 + size 分层）
    • 快排/归并模板
    • 单调栈模板
    • 并查集 + 路径压缩
