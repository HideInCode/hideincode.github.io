大语言模型原理深度解析

一、语言模型基础

1.1 语言模型的目标

语言模型本质上是一个概率模型：给定前文，预测下一个词的概率分布。即：

P(w_t | w_1, w_2, ..., w_{t-1})

1.2 从 N-gram 到神经网络

  • N-gram：统计固定窗口内的词频，数据稀疏，无法长程依赖
  • RNN / LSTM：序列建模，但并行度差、梯度消失/爆炸
  • Transformer：自注意力机制，完全并行，可以捕捉任意距离的依赖

二、Transformer 架构

Transformer 是 LLM 的核心骨架，由编码器（Encoder）和解码器（Decoder）组成，但现代生成式 LLM 大多只使用解码器（Decoder-only）结构，如 GPT 系列。

2.1 整体结构（Decoder-only）

输入 → 词嵌入 + 位置编码 → N 个 Transformer Block → 输出层（线性 + Softmax）

每个 Block 包含：

  • 掩码多头自注意力（Masked Multi-Head Self-Attention）
  • 层归一化（Layer Normalization）
  • 前馈神经网络（Feed-Forward Network, FFN）
  • 残差连接（Residual Connection）

2.2 自注意力机制（Self-Attention）

自注意力允许每个词看到序列中所有其他词，并分配不同的注意力权重。核心公式：

Attention(Q, K, V) = softmax(QK^T / √d_k) V
  • Q（Query）：当前词想要查询的信息
  • K（Key）：序列中每个词的关键信息
  • V（Value）：序列中每个词的实际信息
  • √d_k：缩放因子，防止点积过大导致梯度消失

掩码：Decoder 需要屏蔽未来信息，确保位置 i 只能依赖于小于 i 的位置。

2.3 多头注意力（Multi-Head Attention）

将 Q、K、V 线性投影到 h 个不同的低维空间，分别计算注意力，最后拼接。这允许模型关注来自不同表示子空间的信息。

MultiHead(Q, K, V) = Concat(head_1, ..., head_h) W^O
head_i = Attention(QW_i^Q, KW_i^K, VW_i^V)

2.4 前馈网络（FFN）

通常是两层 MLP，中间使用 ReLU 或 GELU 激活：

FFN(x) = max(0, xW_1 + b_1) W_2 + b_2

2.5 位置编码

Transformer 本身没有处理序列顺序的能力，需要注入位置信息：

  • 绝对位置编码（Sinusoidal）：用正弦/余弦函数生成固定编码
  • 可学习位置编码：将位置编码作为可训练参数
  • RoPE（旋转位置编码）：通过旋转矩阵将位置信息融入注意力，当前主流方案，广泛应用于 LLaMA 等模型

三、训练流程

3.1 预训练（Pre-training）

在海量无标注文本上，使用语言模型目标（预测下一个词）训练模型，让模型学会语言知识、常识、推理能力等。

  • 数据：Common Crawl、书籍、维基百科、代码等（TB 级别）
  • 模型参数：从数百万到数千亿不等
  • 训练策略：自回归（Auto-regressive），即前面提到的 Decoder-only

3.2 指令微调（Supervised Fine-Tuning, SFT）

预训练模型虽然懂得很多知识，但不一定懂得“服从指令”。通过收集指令-回复的高质量数据，对模型进行有监督微调，使其能够理解人类意图并遵循对话格式。

  • 数据样例：{"instruction": "翻译成英文", "input": "你好", "output": "Hello"}
  • 通常只会进行少量 epochs（1-3），过多容易过拟合或灾难性遗忘

3.3 基于人类反馈的强化学习（RLHF）

为进一步对齐人类偏好（如真实性、安全性、帮助性），引入 RLHF 流程：

  1. 收集偏好数据：让模型生成多个回答，由人类标注哪个更好
  2. 训练奖励模型（Reward Model）：用偏好数据训练一个打分模型，模拟人类判断
  3. PPO 强化学习：用奖励模型提供反馈，优化生成策略

RLHF 的本质是用强化学习在奖励模型的引导下微调 LLM，使其行为更符合人类期望。

四、LLM 的核心能力与涌现

4.1 涌现能力（Emergent Abilities）

当模型规模达到一定量级（通常数百亿参数以上），会突然表现出小模型所不具备的能力，如：

  • 少样本学习（Few-shot learning）
  • 复杂推理（Chain-of-Thought, CoT）
  • 世界知识融合
  • 多语言能力

4.2 上下文学习（In-Context Learning）

无需更新模型参数，仅通过在 Prompt 中提供少量示例，模型就能完成对应任务。比如：

请将以下句子翻译为英文：
示例1：苹果 → Apple
示例2：桌子 → Table
你好 →

五、推理与解码策略

文本生成是一个逐个 token 的采样过程，常见策略：

  • 贪婪搜索：每次选概率最大的 token，容易陷入重复
  • 束搜索（Beam Search）：保留概率最高的 k 个候选路径，扩大搜索宽度
  • Top-k 采样：只从概率最高的 k 个 token 中随机选择
  • Top-p（Nucleus）采样：从累积概率超过 p 的最小 token 集合中随机选择
  • 温度（Temperature）：调节概率分布的尖锐程度，温度越高输出越随机，越低越确定

六、主流模型家族

  • GPT 系列（OpenAI）：Decoder-only，从 GPT-1 到 GPT-4，性能不断攀升
  • LLaMA 系列（Meta）：开源，技术报告详实，衍生出了 Alpaca、Vicuna 等
  • ChatGLM（智谱）：国产开源，中英双语
  • Qwen（阿里通义千问）：支持多尺寸和长窗口
  • DeepSeek：以代码和推理见长，训练高效

七、推理优化与部署

大模型推理面临内存和延迟挑战，常用优化手段：

  • KV Cache：缓存历史键值对，避免重复计算
  • 量化（Quantization）：将 FP32 参数压缩为 INT8/INT4，减少内存占用
  • 模型并行（Tensor/Pipeline Parallelism）：将单模型切分到多 GPU
  • FlashAttention：利用 IO 感知算法加速注意力计算
  • 服务框架：vLLM、TGI、llama.cpp 等

八、面试高频追问

问：Transformer 与 LSTM 相比有什么优势？
答：可并行计算，不受序列长度限制（自注意力 O(1) 路径长度），长程依赖建模能力强。

问：为什么 LLM 大多采用 Decoder-only 结构？
答：自回归生成天然适配语言模型目标，训练效率高，且不需要单独的 Encoder 输入。

问：预训练和后训练（SFT/RLHF）分别带来什么？
答：预训练注入知识和基础语言能力；SFT 教会模型遵循指令格式；RLHF 对齐人类偏好。

问：什么是幻觉（Hallucination）？如何缓解？
答：模型生成看似合理但事实性错误的内容。缓解方法：RAG（检索增强）、工具调用、更好的对齐训练。

问：如何处理长上下文？
答：使用 RoPE 扩展、滑动窗口注意力、LongLoRA 等；现有模型已支持 128K 甚至更长的上下文窗口。

理解 LLM 的原理和训练范式，不仅能帮助你用好模型，也能把握这个 AI 时代最核心的技术脉搏。
