Spring

Spring 是 Java 开发者绕不开的框架，读懂它的核心源码能帮你更好地应对面试和线上问题。本文精选了 6 个经典模块的核心代码片段，用大白话拆解 IOC 容器、AOP 切面、事务管理等的内部实现，希望能帮你打通 Spring 的任督二脉。

1. IOC 容器的核心：Bean 的加载过程

ApplicationContext 启动时，最重要的就是加载 BeanDefinition 并创建 Bean。下面是 AbstractApplicationContext.refresh() 方法的骨架，展示了容器初始化的 12 个步骤。

设计要点

  • invokeBeanFactoryPostProcessors 会执行 ConfigurationClassPostProcessor，它负责解析 @Configuration、@ComponentScan 等注解，生成 BeanDefinition。
  • registerBeanPostProcessors 只是注册，不执行，真正的回调发生在 Bean 初始化前后。
  • finishBeanFactoryInitialization 里会遍历 beanDefinitionMap，依次调用 getBean() 完成 Bean 的创建。

2. Bean 的创建过程：从 doGetBean 到 createBean

当调用 getBean() 时，会经历完整的创建或获取流程。下面是 AbstractBeanFactory.doGetBean() 的简化逻辑。

核心套路

  • 单例 Bean 会被缓存在 singletonObjects 中，重复获取直接返回。
  • createBean() 内部调用 doCreateBean()，包含实例化、属性填充、初始化、后置处理等步骤。
  • 循环依赖通过三级缓存解决：singletonFactories、earlySingletonObjects、singletonObjects。

3. AOP 代理的创建：从 @EnableAspectJAutoProxy 到动态代理

AOP 的核心是 AnnotationAwareAspectJAutoProxyCreator，它是一个 BeanPostProcessor，在 Bean 初始化后创建代理对象。

关键点

  • 如果目标类实现了接口，默认使用 JDK 动态代理；否则使用 CGLIB。可通过 proxyTargetClass = true 强制 CGLIB。
  • getAdvicesAndAdvisorsForBean 会扫描所有切面，找到匹配当前 Bean 的通知，再排序生成拦截器链。
  • 最终通过 JdkDynamicAopProxy.invoke() 或 CGLIB 方法拦截执行通知。

4. 声明式事务的实现：@Transactional 的魔法

@Transactional 也是通过 AOP 实现的。核心在于 BeanFactoryTransactionAttributeSourceAdvisor 和 TransactionInterceptor。

传播行为与回滚规则

  • TransactionAttribute 包含了事务传播行为、隔离级别、回滚异常类型等信息。
  • completeTransactionAfterThrowing 会判断异常是否触发回滚，非回滚异常则仍然提交。
  • 多个事务方法嵌套时，TransactionAspectSupport 维护了一个 ThreadLocal 的事务信息栈。

5. Spring MVC 请求分发：DispatcherServlet 的核心调度

DispatcherServlet 是 Spring MVC 的前端控制器，所有请求都会经过它。其核心在 doDispatch() 方法。

请求处理流程

  1. getHandler() 通过 HandlerMapping 列表找到匹配的 HandlerExecutionChain（包含处理器和拦截器）。
  2. HandlerAdapter 负责调用具体的处理器方法，如 RequestMappingHandlerAdapter 会处理 @RequestMapping 注解。
  3. processDispatchResult 包含视图解析，最终将 ModelAndView 渲染为 HTTP 响应。

6. 循环依赖的解决方案：三级缓存

Spring 解决单例 Bean 循环依赖的关键是三级缓存，下面是 DefaultSingletonBeanRegistry 中的核心代码。

为什么需要三级缓存？

  • singletonFactories 存放的是能创建提前引用对象的工厂，可以在 Bean 实例化后、填充属性前将自身暴露出来。
  • A 依赖 B，B 依赖 A：A 实例化后把自己放入三级缓存，然后填充 B；B 实例化过程中发现需要 A，从三级缓存拿到 A 的早期引用；B 完成初始化后放入一级缓存，A 继续填充后也放入一级缓存。
  • 三级缓存同时解决了代理对象的循环依赖，因为工厂可以返回原始对象的代理。
