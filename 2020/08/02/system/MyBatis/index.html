Mybatis

2020-08-02 系统设计 system design 0 评论

Mybatis

  1. 半自动ORM框架，对JDBC进行了封装。
  2. 使用xml/注解进行配置和映射。
  3. 灵活编写SQL，不用与代码耦合。
  4. 兼容各种数据库。
  5. 提供了映射标签。
  6. 但是SQL编写量变大，多表时麻烦；SQL语句以来数据库，不好随便换库。
  7. 不要用RowBounds进行分页，这是个内存分页。

一、基础概念

1.1 MyBatis 是什么？

MyBatis 是一款优秀的持久层框架，它封装了 JDBC 的操作，通过 XML 或注解将接口方法与 SQL 语句映射，支持动态 SQL、结果自动映射和缓存。核心特点是：SQL 完全由开发者掌控，轻量且灵活。

1.2 与 Hibernate 的区别

维度 MyBatis Hibernate
映射方式 手动编写 SQL，结果映射 全自动 ORM，基于注解/Xml 自动生成 SQL
性能调优 直接优化 SQL，粒度细 对 HQL 调优，不透明
学习曲线 低，必须掌握 SQL 高，需理解 Hibernate 状态机
适用场景 复杂报表、高性能要求 标准 CRUD 较多，快速开发

二、核心组件与执行流程

2.1 核心对象

  • SqlSessionFactory：全局工厂，线程安全，负责创建 SqlSession
  • SqlSession：一次数据库会话，非线程安全，用完即关
  • MappedStatement：封装 SQL 映射信息（SQL 语句、参数类型、返回类型等）
  • Executor：执行器，负责 SQL 的执行（Simple、Reuse、Batch）

2.2 执行流程

  1. 通过 SqlSessionFactoryBuilder 读取配置文件构建工厂
  2. 由工厂打开 SqlSession
  3. 调用 getMapper 获取接口的动态代理对象
  4. 代理调用方法 → 根据接口全限定名+方法名找到 MappedStatement
  5. Executor 执行 SQL 并映射结果

三、配置文件与映射

3.1 全局配置（mybatis-config.xml）

主要配置：别名、环境（数据源）、映射文件、插件、设置

<configuration>
  <settings>
    <setting name="cacheEnabled" value="true"/>
    <setting name="lazyLoadingEnabled" value="true"/>
    <setting name="mapUnderscoreToCamelCase" value="true"/>
  </settings>
  <environments default="development">
    <environment id="development">
      <transactionManager type="JDBC"/>
      <dataSource type="POOLED">
        <property name="driver" value="com.mysql.cj.jdbc.Driver"/>
        <property name="url" value="jdbc:mysql://localhost:3306/db"/>
        <property name="username" value="root"/>
        <property name="password" value="123456"/>
      </dataSource>
    </environment>
  </environments>
  <mappers>
    <mapper resource="mapper/UserMapper.xml"/>
  </mappers>
</configuration>

3.2 映射文件（Mapper.xml）

<mapper namespace="com.example.mapper.UserMapper">
  <select id="findById" resultType="User" parameterType="long">
    SELECT * FROM users WHERE id = #{id}
  </select>
</mapper>

#{} 与 ${} 区别：#{} 是预编译占位符，防止 SQL 注入；${} 直接拼接字符串，仅在需要动态表名或列名时使用，必须做好安全过滤。

四、动态 SQL

MyBatis 的动态 SQL 是其最强大的特性之一，常用标签：

  • **<if>**：条件判断

  • **<choose> <when> <otherwise>**：多条件选择

  • **<where>**：自动处理 WHERE 关键字及多余 AND/OR

  • **<set>**：UPDATE 时自动处理 SET 逗号

  • **<foreach>**：遍历集合，生成 IN 条件或批量插入

  • **<trim>**：自定义字符串截取

五、缓存机制

5.1 一级缓存（Local Cache）

  • 作用域：SqlSession 级别，默认开启，无法跨会话共享
  • 失效条件：执行更新操作、手动 clearCache()、不同 SqlSession

5.2 二级缓存

  • 作用域：Mapper 级别（namespace），跨 SqlSession 共享

  • 开启步骤：全局配置 cacheEnabled=true，Mapper 中添加 <cache/>，实体类实现 Serializable

  • 刷新时机：该 namespace 下的任何 insert/update/delete 都会清空缓存

    ...

  • 风险：脏读和内存占用，谨慎在经常修改的表中使用二级缓存

六、高级特性

6.1 结果映射与关联查询

  • **<resultMap>**：自定义字段映射、处理一对多/多对一

  • **<association>**：多对一、一对一关联

  • **<collection>**：一对多关联

6.2 插件机制

MyBatis 允许通过插件拦截以下四大对象的调用：

  • Executor (执行器方法)
  • ParameterHandler (参数处理)
  • ResultSetHandler (结果集处理)
  • StatementHandler (预编译 SQL 处理)

分页插件（如 PageHelper）就是利用 Executor 的拦截实现。

七、MyBatis-Plus 基础

MyBatis-Plus 是 MyBatis 的增强工具，简化 CRUD：

public interface UserMapper extends BaseMapper<User> { }

// 使用
List<User> users = userMapper.selectList(new LambdaQueryWrapper<User>()
    .eq(User::getStatus, 1)
    .like(User::getName, "张"));

优势：内置通用 Mapper、条件构造器、分页、乐观锁，减少重复代码。

八、常见问题排查

8.1 找不到 Mapper 映射

  • 检查 namespace 与接口全限定名是否一致
  • 检查映射文件路径是否在配置中指定
  • 在 Spring Boot 中使用 @MapperScan 或每个接口加 @Mapper

8.2 SQL 执行慢

  • 开启 SQL 日志：logging.level.包名.mapper=debug
  • 分析索引是否命中
  • 检查是否大量使用了 ${} 导致 SQL 硬解析
  • 关联查询是否产生笛卡尔积

8.3 内存溢出

  • 大数据量查询未分页
  • 开启了二级缓存，缓存体积过大
  • 使用流式查询解决：@Options(resultSetType = ResultSetType.FORWARD_ONLY, fetchSize = Integer.MIN_VALUE)

8.4 #{} 和 ${} 混用导致注入

严禁在接收用户输入的场合使用 ${}。如果确实需要动态列名，应在服务层做白名单校验。

九、面试高频问答

问：MyBatis 是如何进行预编译的？
使用 #{} 时，MyBatis 通过 PreparedStatement 设置参数，JDBC 驱动负责预编译，有效防止 SQL 注入。

问：MyBatis 的分页原理？
逻辑分页：将所有记录查询到内存中再截取；物理分页：使用数据库的分页语句（如 LIMIT），可配合 PageHelper 插件拦截 SQL 追加分页参数。

问：如何实现批量插入？
使用 <foreach> 生成多值 SQL，或使用 SqlSession 的 BATCH 模式，一次性 flushStatements()。

INSERT INTO users (name, age) VALUES
<foreach collection="list" item="user" separator=",">
  (#{user.name}, #{user.age})
</foreach>

问：MyBatis 延迟加载如何实现？
基于代理对象，当访问关联属性时触发二次查询。配置 lazyLoadingEnabled=true 和 aggressiveLazyLoading=false（MyBatis 3.4.1 后默认不激进加载）。

熟练掌握 MyBatis 的 SQL 控制与高级映射，能让你在复杂业务中游刃有余。
