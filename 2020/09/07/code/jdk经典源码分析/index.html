JDK源码分析

2020-09-07 源码 code 0 评论

JDK

阅读 JDK 源码是提升 Java 内功的必经之路。本文精选了 5 段经典代码，涵盖集合框架和并发编程的核心实现，附详细注释，帮助你看懂那些优雅的设计。

1. HashMap 的 put 方法

HashMap 是面试必问的容器，其 put 方法涵盖了寻址、哈希碰撞、红黑树化和扩容等核心逻辑。

设计亮点

  • (n - 1) & hash 代替取模运算，效率更高。
  • 链表转红黑树的阈值为 8，兼顾查找效率与内存开销。
  • 扩容时重新计算每个节点的位置，避免所有元素重新哈希。

2. ArrayList 的扩容机制

ArrayList 底层是数组，插入元素时可能触发扩容，下面展示其核心方法。

设计亮点

  • 1.5 倍扩容在空间利用率和复制开销间取得平衡。
  • 使用 Arrays.copyOf 直接创建新数组并复制，内部调用 System.arraycopy，高效且安全。

3. ConcurrentHashMap 的 size 方法

ConcurrentHashMap 在 JDK 1.8 中放弃了分段锁，改用 CAS + synchronized。其 size 方法通过乐观策略避免了长时间的锁竞争。

设计思路

  • 使用 baseCount 保存全局计数，非竞争时通过 CAS 更新 baseCount。
  • 当 CAS 失败（并发冲突），将增量存入 CounterCell 数组的一个随机槽中。
  • 最终统计时累加 baseCount 和所有 CounterCell 的值，从而在无锁状态下获得近似准确或精确的计数。

为什么不用 AtomicLong？
单个 AtomicLong 在高并发下会成为热点，CounterCell 将竞争分散到多个槽，提升性能。

4. ThreadPoolExecutor 的线程池状态与生命周期

线程池的 runState 和 workerCount 被巧妙保存在一个 AtomicInteger 中：高 3 位表示状态，低 29 位表示线程数。

状态流转
RUNNING → SHUTDOWN（不接受新任务，处理已有任务） → STOP（不接受新任务，中断正在执行的任务） → TIDYING（所有任务终止，线程数为 0） → TERMINATED（terminated() 钩子方法已执行）。

设计亮点
用一个变量同时表示状态和数量，配合 CAS 操作，实现无锁的线程池状态管理。

5. Integer 的缓存机制

Integer.valueOf() 会缓存 -128 到 127 之间的整数，这是享元模式（Flyweight）的经典应用。

使用建议

  • 业务对象尽量用 equals 比较内容，不要用 == 比较引用（虽然缓存范围内的 == 可能为 true，但超出范围就不可靠了）。
  • 对于频繁使用的数值，缓存避免了重复创建对象，节省内存。
